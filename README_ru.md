# keyhunt (RU)

Язык:
- Русский (этот файл)
- English: ./README.md

Инструмент для поиска приватных ключей для криптовалют на кривой secp256k1.

Ссылки:
- Форум/тред: https://bitcointalk.org/index.php?topic=5322040.0

## Быстрые примеры для пазлов

Puzzle 66 (режим `address`):

```bash
./keyhunt -m address -f tests/66.txt -b 66 -l compress -R -q -s 10 -t 8
```

Puzzle 66 (режим `rmd160`):

```bash
./keyhunt -m rmd160 -f tests/66.rmd -b 66 -l compress -R -q -s 10 -t 8
```

Puzzle 125 (режим `bsgs`):

```bash
./keyhunt -m bsgs -f tests/125.txt -b 125 -R -q -s 10 -t 8 -k 512
```

## Оптимальные параметры BSGS: `-k` и `-n` (до 1024GB RAM)

Важно:
- В режиме BSGS ключ `-n` задаёт параметр `N` (можно десятичный или `0xHEX`).
- `N` должен иметь **точный** квадратный корень.
- `M = sqrt(N)` должен делиться на **1024**.
- `-k` ускоряет, но увеличивает потребление RAM и размер файлов `.blm/.tbl`.
- Рекомендуется использовать степени двойки для `-k` и `-n`.

База проекта:
- По умолчанию `-n = 0x100000000000`.
- При `-k 4096` и дефолтном `-n` потребление памяти порядка ~64GB.

Чтобы масштабировать потребление RAM **выше ~64GB** и при этом держать `-k` в «безопасном/дефолтном» диапазоне (<= 4096), увеличивайте `-n`:

| Целевая RAM | Рекомендуемый `-k` | Рекомендуемый `-n` |
| --- | ---: | --- |
| 8 GB | 512 | 0x100000000000 |
| 16 GB | 1024 | 0x100000000000 |
| 32 GB | 2048 | 0x100000000000 |
| 64 GB | 4096 | 0x100000000000 |
| 128 GB | 4096 | 0x400000000000 |
| 256 GB | 4096 | 0x1000000000000 |
| 512 GB | 4096 | 0x4000000000000 |
| 1024 GB | 4096 | 0x10000000000000 |

Всегда сверяйтесь с выводом программы (она печатает размеры bloom/table) и подбирайте значения под свою машину.
